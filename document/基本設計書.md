# 基本設計書（Architecture & Design）

## 1. システム構成・画面遷移
本アプリは外部サーバーと通信を行わない**シングルページアプリケーション（SPA）**として構築し、HTML/CSS/JavaScriptを1ファイル（`index.html`）に集約します。JavaScriptによって表示するコンテナ（`div`）を切り替えることで画面遷移を実現します。

```
[初回起動] ──> 設定画面（screen-setup）
                 │ (初回保存後)
                 ▼
[通常起動] ──> 子供メニュー（screen-menu） ──(バトル開始)──> バトル画面（screen-battle）
                 │
                 ├─(保護者ボタン)─> パスコード入力（screen-gate）
                 │                                    │ (正解)
                 │                                    ▼
                 └───────────────────────────────> 保護者画面（screen-parent）
```

## 2. 画面一覧
1. **設定画面 (`screen-setup`):** 初回起動時のみ表示。入学年月日と初期パスコードを入力。
2. **子供メニュー画面 (`screen-menu`):** 学年・レベル・難易度選択、教科別バトル開始ボタン、保護者画面へのゲートボタンを配置。
3. **パスコードゲート画面 (`screen-gate`):** 保護者画面に入るための数字入力画面。
4. **保護者画面 (`screen-parent`):** お子様の教科別正解数（スタッツ）の確認、および入学年月日の再設定。
5. **バトル画面 (`screen-battle`):** プレイヤー・敵のHPバー、問題文、4択の回答ボタンを表示。

## 3. データ構造（データ設計）
ブラウザの `localStorage` 内にキー名 `learning_quest_data` として、以下のJSONオブジェクトを一元管理します。

```json
{
  "admissionDate": "2026-04-01",
  "passcode": "1234",
  "level": 1,
  "score": 0,
  "stats": {
    "math": 0,
    "japanese": 0,
    "life": 0
  },
  "currentDifficulty": "easy"
}
```

## 4. バトルシステム・パラメータ設計

| 選択難易度 | 出題内容 | プレイヤー受けるダメージ | 敵に与えるダメージ倍率 |
| :--- | :--- | :--- | :--- |
| **かんたん** | 対象時期の基礎問題 | 0（ノーダメージ） | 1.0倍（基礎ダメージ 34） |
| **ふつう** | 対象時期の標準問題 | 20ダメージ | 1.0倍（基礎ダメージ 25） |
| **むずかしい** | 対象時期の応用・予習 | 34ダメージ | **2.0倍（大ダメージ）** |
